Academic Library Book Management System - Database Schema

This document describes the MongoDB/Mongoose schemas currently used in the project.
Collections: users, books, issues

============================================================
1) USER SCHEMA (Collection: users)
============================================================

| Field Name | Data Type | Constraints | Required | Description |
|---|---|---|---|---|
| _id | ObjectId | Auto-generated, Primary Identifier | Yes (auto) | Unique document identifier created by MongoDB. |
| name | String | trim: true | Yes | Full name of the user. |
| email | String | unique: true, lowercase: true, trim: true | Yes | User login email; must be unique. |
| password | String | Stored as hashed value (application logic) | Yes | User password hash for authentication. |
| role | String | enum: ['student', 'faculty', 'librarian', 'admin'], default: 'student' | No (has default) | Role that controls permissions in the system. |
| createdAt | Date | Auto-generated by timestamps | Yes (auto) | Date/time when the user record was created. |
| updatedAt | Date | Auto-updated by timestamps | Yes (auto) | Date/time when the user record was last updated. |

============================================================
2) BOOK SCHEMA (Collection: books)
============================================================

| Field Name | Data Type | Constraints | Required | Description |
|---|---|---|---|---|
| _id | ObjectId | Auto-generated, Primary Identifier | Yes (auto) | Unique document identifier created by MongoDB. |
| title | String | trim: true | Yes | Title of the book. |
| author | String | trim: true | Yes | Author name of the book. |
| isbn | String | unique: true, trim: true | Yes | International Standard Book Number; unique per book entry. |
| category | String | trim: true | Yes | Subject/category of the book (e.g., Computer Science). |
| publisher | String | trim: true | Yes | Publisher name of the book. |
| totalCopies | Number | min: 0 | Yes | Total number of copies owned by the library. |
| availableCopies | Number | min: 0 | Yes | Number of copies currently available for issue. |
| shelfLocation | String | trim: true | Yes | Physical shelf/rack location in the library. |
| createdAt | Date | Auto-generated by timestamps | Yes (auto) | Date/time when the book record was created. |
| updatedAt | Date | Auto-updated by timestamps | Yes (auto) | Date/time when the book record was last updated. |

============================================================
3) ISSUE SCHEMA (Collection: issues)
============================================================

| Field Name | Data Type | Constraints | Required | Description |
|---|---|---|---|---|
| _id | ObjectId | Auto-generated, Primary Identifier | Yes (auto) | Unique document identifier created by MongoDB. |
| userId | ObjectId | ref: 'User' | Yes | Reference to the user who requested/borrowed the book. |
| bookId | ObjectId | ref: 'Book' | Yes | Reference to the requested/issued book. |
| issueDate | Date | default: Date.now | No (has default) | Date/time when issue entry is created/issued. |
| dueDate | Date | - | Yes | Date/time when the book should be returned. |
| returnDate | Date | default: null | No (nullable/default) | Date/time when the book is returned; null if not returned yet. |
| status | String | enum: ['Pending', 'Issued', 'Returned', 'Overdue'], default: 'Pending' | No (has default) | Current issue lifecycle state. |
| createdAt | Date | Auto-generated by timestamps | Yes (auto) | Date/time when the issue record was created. |
| updatedAt | Date | Auto-updated by timestamps | Yes (auto) | Date/time when the issue record was last updated. |

============================================================
Relationships Summary
============================================================

- User (1) to Issue (many): One user can have multiple issue records.
- Book (1) to Issue (many): One book can appear in multiple issue records over time.
- Issue links User and Book via userId and bookId references.
